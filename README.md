# WARMOPS Dashboard

WARMOPS Dashboard is a role based operations management dashboard designed to help organizations manage internal work, monitor activities, track projects, control user permissions, and view operational indicators from one centralized interface.

This repository is created as a showcase for the project. It includes the project overview, live demo link, demo accounts, screenshots, and feature documentation. The source code is private to protect the implementation.

## Project Overview

WARMOPS Dashboard provides a structured internal system for managing operations through four user roles: Admin, Manager, Employee, and Reader.

Each role has different access levels based on its responsibility. The main goal of the system is to organize internal workflows, improve visibility, and support faster decision making through dashboards, reports, notifications, and KPI cards.

## User Roles

### Admin

The Admin has full access to the system. This role can manage users, permissions, projects, tasks, reports, notifications, and system settings. The Admin dashboard provides a complete view of system activity and operational performance.

### Manager

The Manager can manage assigned projects, follow team progress, review tasks, monitor reports, and track performance indicators. This role is focused on supervision and operational follow up.

### Employee

The Employee can view assigned tasks, update task progress, receive notifications, and follow work related updates. This role is focused on daily execution and task completion.

### Reader

The Reader can view dashboards, reports, activities, and system information without creating, editing, or deleting data. This role is designed for users who need visibility without direct control.

## Main Screens

### Admin Dashboard

The Admin dashboard gives a high level overview of the system. It includes KPI cards, charts, recent activity, notifications, and quick access to important sections.

### Projects

The projects section allows users to view and manage project information, track project status, and monitor overall progress.

### Tasks

The tasks section helps organize assigned work, task status, priorities, and progress updates.

### Users and Permissions

The users section allows the Admin to manage accounts and assign roles. The system separates access between Admin, Manager, Employee, and Reader to keep permissions controlled.

### Reports

The reports section summarizes operational data and presents it in a clear way to support decision making.

### Notifications

The notifications section helps users stay updated with important actions, alerts, and system events.

## Purpose of the Project

The purpose of WARMOPS Dashboard is to demonstrate how an internal operations system can improve work visibility, organize tasks and projects, control user permissions, and provide useful indicators for decision making.

The project was built as a prototype to practice dashboard development, role based access control, operational data visualization, interface design, and workflow organization.

## What I Learned

Through this project, I practiced designing a dashboard structure that separates users based on their responsibilities. I also worked on organizing operational data, building clear interfaces, managing permissions, and presenting information in a way that helps users understand system activity quickly.

The project helped me understand that a dashboard is not only about charts. It is about showing the right information to the right user at the right time.
